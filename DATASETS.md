# RA-SpMM Benchmark Datasets

The RA-SpMM evaluation suite consists of **51 sparse graphs** (26 real-world + 25 procedurally generated synthetic) used in the FGCS paper. To keep the GitHub repository lightweight (~2 MB), the actual graph data is hosted on Zenodo as a single archive.

## Zenodo record

- **DOI**: `10.5281/zenodo.19903312`
- **URL**: `https://zenodo.org/records/19903313`
- **Archive**: `ra_spmm_data_v1.tar.gz` (~1.9 GB compressed; ~6.9 GB extracted)
- **SHA-256**: `eedcdc6285ce33a3af4e18ea8bd14d73cb43c2582c9ae362ee6bdc980f0a604f`
- **License**: CC-BY 4.0 for the synthetic graphs; SNAP / OGB / PyG terms for the real graphs (see ATTRIBUTIONS below).

## One-step download and extract

After cloning this repository, run from the repository root:

```bash
# Download the Zenodo bundle (~1.9 GB compressed)
wget https://zenodo.org/records/19903313/files/ra_spmm_data_v1.tar.gz

# Extract; --strip-components=1 drops the top-level wrapper so files
# land in ./datasets/ and ./fgcs_results/ where the manifests expect them
tar -xzf ra_spmm_data_v1.tar.gz --strip-components=1

# Sanity check
python -c "
import json, os
m = json.load(open('paper_datasets.json'))['datasets']
miss = [d['name'] for d in m if not os.path.exists(d['path'])]
print('Missing real graphs:', miss or 'none')
"
# Expected: Missing real graphs: none
```

Or use the provided helper script: `bash scripts/fetch_datasets.sh` (Linux/macOS) or `python scripts/fetch_datasets.py` (cross-platform).

## What the bundle contains

```
ra_spmm_data_v1/
├── README.md                              short reproduction note
├── datasets/                              26 real-world graphs (~6.0 GB)
│   ├── *.txt                              raw SNAP edge lists (17 graphs)
│   ├── amazon-{computers,photo}.npz       custom-NPZ format used by ra_real_graph_eval.py
│   └── gnn/exports/                       11 PyG/OGB graphs as scipy CSR NPZ
│       ├── amazon-{computers,photo}.npz   (also in gnn-NPZ format for gnn_bench scripts)
│       ├── Cora.npz, CiteSeer.npz, Flickr.npz, PPI.npz
│       ├── ogbn-{arxiv,products,proteins}.npz
│       ├── Reddit.npz, Yelp.npz
└── fgcs_results/                          25 synthetic graphs (~888 MB)
    ├── paper_synthetic_datasets.json      25-entry manifest
    ├── paper_combined_datasets.json       51-entry combined manifest
    └── synthetic/
        ├── synth_community_nc{10,50,100,500,1000}.npz
        ├── synth_dense_small_d{30,50,70,90,120}.npz
        ├── synth_mixed_v{1..5}.npz
        ├── synth_sparse_skewed_cv{1.5,2,2.5,3,4}.npz
        └── synth_sparse_uniform_d{3,5,8,12,18}.npz
```

After extraction, the repo's `paper_datasets.json` and `paper_combined_datasets.json` resolve every entry against on-disk files, and `python ra_router_parity_test.py` should print `PARITY OK 192/192`.

## Subset reproduction (smaller download)

If you only need the SpMM-level reproduction and not the full GNN end-to-end benchmark, the `datasets/gnn/exports/` subset is optional except for these graphs that are referenced from `paper_datasets.json`:

- `ogbn-arxiv.npz` (3 MB)
- `Yelp.npz` (115 MB)
- The four Mixed/Irregular graphs: `Cora.npz`, `CiteSeer.npz`, `Flickr.npz`, `PPI.npz` (~7.5 MB)

The 25 synthetic graphs can also be **regenerated locally** in ~30 seconds without downloading them, using the deterministic-seed generator that ships in this repo (`fgcs_results/synthetic_generate.py`, seeds 10000–50005).
